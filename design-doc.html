NPS Trailhead Outfitters Design Document

Planning page for a National Park Service inspired outdoor gear storefront. The goal is to help visitors browse products, add items to a cart, and review their order.

Wireframes Colors Typography Technical Plan

Site Purpose

NPS Trailhead Outfitters will be a simple storefront for national park inspired outdoor gear. Users will be able to browse product cards, add items to a cart, and review selected products before checkout. The design should feel outdoorsy, readable, and easy to use on both mobile and desktop screens.

Color Palette

The palette uses forest green and warm earth tones to connect the site to national parks, trail signs, trees, sunlight, and outdoor equipment. The colors are chosen to keep text readable against light backgrounds.

Primary #2F5D50 Forest green
Secondary #C97B3D Canyon clay
Accent #F2C14E Trail marker yellow
Background #F7F4EC Warm off-white
Text #24312E Dark pine charcoal

Typography

Heading Font

Georgia will be used for headings. It has a classic outdoor guidebook feeling and gives the site a more planned look than a default sans-serif heading.

Body Font

Arial will be used for body text. It is easy to read, widely available in browsers, and works well for product cards, forms, buttons, and cart details.

The final website will avoid Times New Roman as the default page font and will keep text sizes consistent for readability.

Technical Planning

Planned Pages / Views

  • Main page: store header, navigation, featured products, product cards, and add-to-cart buttons.
  • Cart page: selected products, item quantities, subtotal, total, and checkout-style order form.

Planned JavaScript Modules

main.js

Starts the storefront, loads product data, connects event listeners, and updates page content.

data.js

Stores or loads product data, including product names, prices, categories, descriptions, and image paths.

render.js

Creates product cards, cart item rows, quantity displays, subtotal amounts, and cart messages.

storage.js

Handles saving, loading, updating, and removing cart items using localStorage.

Planned Data Structure

The store data will be stored as JSON-style product objects. Each product will include an id, name, category, price, short description, image path, and stock status. 

{
  "id": "product01",
  "name": "NPS Trail Pack",
  "category": "Backpacks",
  "price": 64.99,
  "description": "A day pack for national park hikes and outdoor trips.",
  "image": "images/trail-pack.webp",
  "inStock": true
}

Planned localStorage Feature

The website will use localStorage to save the user's shopping cart. When a user selects an add-to-cart button, the product id and quantity will be stored in an array named cartItems. The Cart page will read that saved list, display the selected products, and calculate the order total.
